# Skill: Technical Analysis (Dev Mode)

## Objective
Transform a product task into a detailed technical implementation plan.

This analysis must follow:
- docs/architectural_standard.md
- docs/stack_definition.md
- docs/test_standard.md.

---

## Required Output Structure

### 1. Domain Modeling
- Entities
- Value Objects
- Responsibilities
- Relationships

### 2. Database Modeling
- Tables
- Fields
- Constraints
- Indexes
- Foreign keys

### 3. Architectural Breakdown
- Ordered implementation tasks

### 4. API Proposal
- Endpoints
- Request/Response format

### 5. Test Strategy
- Unit tests
- Integration tests
- Edge cases

### 6. Risks & Tradeoffs

---

## Persistence Rule

Save output to:
/analysis/app_name/task_name/task_1.md
...
/analysis/app_name/task_name/task_n.md
